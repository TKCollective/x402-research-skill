What you get back

Every paid response includes the verified answer plus a signed receipt. The receipt is a JWS over a canonical hash of the question, the answer, the sources cited, the model used, and the on-chain payment transaction. Anyone with our public JWKS can verify the receipt offline — no AgentOracle service required.

This is the foundation for the v0.2 receipt spec we are filing as an IETF Internet-Draft in early June 2026.

Why x402

x402 is the HTTP-native micropayments standard. The protocol is simple: an agent makes a request, the server responds 402, the agent signs an EIP-3009 USDC payment authorization, the request settles on chain, the resource is delivered. No accounts, no API keys, no subscriptions, no rate-limit drama.

AgentOracle uses the x402 v2 wire format with the bazaar discovery extension. Settles on Base mainnet (eip155:8453) via the Coinbase CDP facilitator.

Indexed and discoverable
